# 顧客分析レポート

## 1. 分析概要

### 1.1 分析目的

スーパーマーケットの顧客データを活用し、**データドリブンなマーケティング施策**を立案・実行するための分析基盤を構築する。

### 1.2 分析フレームワーク

| Step | 観点 | 分析手法 | 答える問い |
|------|------|---------|-----------|
| 1 | **誰に** | RFM分析・デシル分析 | どの顧客層にアプローチすべきか？ |
| 2 | **なぜ** | 行動クラスター分析 | その顧客はなぜその行動をとるのか？ |
| 3 | **いつ** | リピートタイミング分析 | いつ介入すれば離反を防げるか？ |
| 4 | **何を** | クロス分析 | 具体的に何を提案すべきか？ |

### 1.3 データソース

**方式**: ダッシュボードのデータソースにSELECTクエリを設定（CREATE TABLEせず、Goldテーブルを直接参照）

| データソース名 | ベーステーブル | 粒度 | 追加カラム |
|--------------|--------------|------|-----------|
| 顧客分析マート | `gd_users` | 顧客1人1レコード | decile_rank, recency_status, retention_campaign |
| トランザクション分析マート | `gd_order_items` | 注文明細1行1レコード | order_dayofweek_name, time_period |

> **クロスフィルタリング**: 同一データソースを使用することで、ダッシュボード上で全グラフが連動
>
> **SELECTクエリ**: 05_dashboard_mart.py に記載のクエリをコピペして使用

## 4. 課題の特定と深掘り

### 4.1 最優先課題：時短族の離反リスク

**発見**: 時短族は来店頻度が最も高い（2-3日サイクル）が、離反も最速（10日で離反）

**なぜ深刻か**:
- 来店頻度: 週3回以上 → 離反時の売上損失が大きい
- 離反臨界点: 10日（他クラスターの半分以下）
- 要介入ステータス（4-7日経過）の顧客が約20%存在

### 4.2 クラスター別離反リスク

| クラスター | 自然サイクル | 離反臨界点 | 介入タイミング | 深刻度 |
|-----------|-------------|-----------|---------------|--------|
| 平日夕方の時短族 | 2-3日 | **10日** | 4-7日経過時 | 最高 |
| 夜型値引きハンター | 3-5日 | 14日 | 6-10日経過時 | 高 |
| カテゴリジャンキー | 5-7日 | 14日 | 8-10日経過時 | 中 |
| 週末まとめ買い族 | 7日 | 21日 | 8-14日経過時 | 中 |
| 健康・こだわり層 | 7-10日 | 21日 | 11-14日経過時 | 低 |

## 5. 施策シナリオ（優先度順）

### 【最優先】シナリオ1: 時短族の緊急リテンション

**対象者の特定（ダッシュボードでのフィルタ条件）**:
```
behavior_cluster_name = '平日夕方の時短族'
AND recency_status = '要介入'（4-7日経過）
```

**なぜこの施策か（データ根拠）**:
- 時短族の即食系商品購買率: **90%**
  - 内訳: 惣菜30% + 冷凍食品30% + パン30%
- 来店時間帯: 平日16-18時に集中
- → **即食系セット + 夕方限定**が最も効果的

**施策内容**:
| 項目 | 内容 |
|------|------|
| オファー | 「本日限定！惣菜・冷凍食品セット500円」 |
| 配信タイミング | 平日15時（来店ピーク1時間前） |
| チャネル | プッシュ通知 or LINE |
| 有効期限 | 当日18時まで |

**KPI設計**:
| 指標 | 測定方法 | 目標値 |
|------|---------|--------|
| 復帰率 | 施策配信後7日以内の来店有無 | 30% |
| 即食系購買率 | 復帰者の惣菜+冷凍+パン購買率 | 85%以上 |
| 施策ROI | (復帰売上 - クーポン原価) / 配信コスト | 200%以上 |

**効果測定の手順**:
1. 施策前: 対象者リストをエクスポート（user_id一覧）
2. 施策実行: MAツールでクーポン配信
3. 施策後7日: 同じuser_idの来店有無を確認
4. 比較: 配信群 vs 非配信群の復帰率を比較

---

### 【優先度2】シナリオ2: ロイヤル顧客 × 健康・こだわり層の深耕

**対象者の特定（ダッシュボードでのフィルタ条件）**:
```
rfm_segment = 'ロイヤル顧客'
AND behavior_cluster_name = '健康・こだわり層'
```

**なぜこの施策か（データ根拠）**:
- 健康層のオーガニック商品購買率: **84%**
  - 内訳: 野菜28% + 乳製品28% + 調味料28%（全てis_organic対象）
- ロイヤル × 健康層 = 高LTV層がプレミアム商品を指名買い
- → **プレミアム商品の先行案内**で客単価向上

**施策内容**:
| 項目 | 内容 |
|------|------|
| オファー | 新着オーガニック商品の先行案内 + 試食会招待 |
| 配信タイミング | 週末3日前（木曜） |
| チャネル | メール or アプリ |
| 特典 | 年間購買額に応じた特別割引（5-15%） |

**KPI設計**:
| 指標 | 測定方法 | 目標値 |
|------|---------|--------|
| 開封率 | メール開封 / 配信数 | 40%以上 |
| 客単価向上 | 施策前後1ヶ月の平均客単価比較 | +15% |
| 離反率低下 | 施策後3ヶ月の離反率 | -20% |

---

### 【優先度3】シナリオ3: 新規顧客 × 週末まとめ買い族の定着

**対象者の特定（ダッシュボードでのフィルタ条件）**:
```
customer_type = '新規'
AND behavior_cluster_name = '週末まとめ買い族'
```

**なぜこの施策か（データ根拠）**:
- 週末まとめ買い族のバスケットサイズ: **15-30点**（他クラスターの3-5倍）
- 来店曜日: 土日に100%集中
- 重い荷物（飲料、米、洗剤）が多い
- → **配送サービス無料案内**で2回目来店を促進

**施策内容**:
| 項目 | 内容 |
|------|------|
| オファー | 「初回配送無料」+ 「次回週末ポイント2倍」 |
| 配信タイミング | 初回購入後3日以内 |
| チャネル | メール + アプリ |
| 有効期限 | 次の週末まで |

**KPI設計**:
| 指標 | 測定方法 | 目標値 |
|------|---------|--------|
| 2回目来店率 | 初回購入後30日以内の再来店 | +25% |
| 配送サービス加入率 | 配送サービス利用開始率 | 15% |
| 週末来店継続率 | 3ヶ月後も週末に来店している率 | 60% |

---

### 【優先度4】シナリオ4: 離脱リスク × 夜型値引きハンターの復帰

**対象者の特定（ダッシュボードでのフィルタ条件）**:
```
rfm_segment = '離脱リスク'
AND behavior_cluster_name = '夜型値引きハンター'
```

**なぜこの施策か（データ根拠）**:

  - 内訳: 惣菜25% + パン25% + 冷凍食品25%
- 来店時間帯: 19-21時（閉店前値引きタイム）
- → **夜限定タイムセール通知**で復帰を促進

**施策内容**:
| 項目 | 内容 |
|------|------|
| オファー | 「今夜19時〜 惣菜全品30%OFF」 |
| 配信タイミング | 当日18時 |
| チャネル | プッシュ通知 |

**KPI設計**:
| 指標 | 測定方法 | 目標値 |
|------|---------|--------|
| 復帰率 | 施策後14日以内の来店 | 25% |
| 夜間来店率 | 復帰者の19-21時来店率 | 80%以上 |

## 6. 施策優先度マトリクス

| 優先度 | シナリオ | 対象者構成比 | インパクト根拠 | 実施難易度 |
|--------|---------|-------------|---------------|-----------|
| 1 | 時短族リテンション | 10%の約20%=2% | 週3来店×離反=月12回分損失 | 低（クーポン配信のみ） |
| 2 | ロイヤル×健康層深耕 | 上位10%×5%=0.5% | LTV最上位層、離反時の損失極大 | 中（特別施策必要） |
| 3 | 新規×週末族定着 | 新規の10% | 大量購入層の定着=長期売上確保 | 低（自動メール可） |
| 4 | 離脱リスク×夜型復帰 | 離脱リスクの5% | 復帰率は低いが試す価値あり | 低（通知のみ） |

**優先度の判断基準**:
```
インパクト = 対象者数 × 来店頻度 × 客単価 × 改善率
```

## 7. 次のアクション

### 7.1 施策実行フロー

```
1. ダッシュボードで対象者を特定（フィルタ条件を適用）
   ↓
2. 顧客分析ページで対象者数を確認
   ↓
3. 施策対象者リストをエクスポート（user_id一覧）
   ↓
4. MAツールにインポート
   ↓
5. 施策実行（クーポン配信等）
   ↓
6. 効果測定（KPIをダッシュボードで確認）
```

### 7.2 定期モニタリング項目

| 頻度 | 確認項目 | ダッシュボードページ | アクション |
|------|---------|-------------------|-----------|
| 日次 | 日別売上推移 | 経営サマリー | 異常値の早期発見 |
| 週次 | リピートステータス分布 | 顧客分析 | 要介入顧客への施策実行 |
| 週次 | 新規/既存売上比率 | 経営サマリー | 新規獲得施策の効果確認 |
| 月次 | RFMセグメント変動 | 顧客分析 | セグメント移動の追跡 |
| 月次 | デシル構成比 | 顧客分析 | 上位顧客の維持確認 |
