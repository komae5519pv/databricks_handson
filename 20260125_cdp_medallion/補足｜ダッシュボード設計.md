# ダッシュボード帳票設計

## 1. ダッシュボード概要

**ダッシュボード名**: CDP顧客分析ダッシュボード

### ページ構成（3タブ）

| タブ | ページ名 | 問い | 見る人 | データソース |
|-----|---------|------|--------|------------|
| 1 | 経営サマリー | 今どうなってる？ | 店長・経営層 | `transaction_analysis` |
| 2 | 売上分析 | なぜそうなってる？ | マーケ・分析担当 | `transaction_analysis` |
| 3 | 顧客分析 | 誰がそうしてる？ | CRM・施策担当 | `customer_analysis` |

---

## 2. データソース設定

### 設定手順

1. **Dashboards → Create Dashboard**
2. **Data → Add data → Write custom SQL**
3. データソース名を入力
4. 05_dashboard_mart.py のクエリをコピペ
5. カタログ・スキーマ名を自分の環境に変更

### データソース一覧

| データソース名 | クエリ元 | 粒度 |
|--------------|---------|------|
| `transaction_analysis` | 05_dashboard_mart.py | 注文明細1行1レコード |
| `customer_analysis` | 05_dashboard_mart.py | 顧客1人1レコード |

## 3. ページ1: 経営サマリー

**目的**: 3秒で現状把握、即座に意思決定

**データソース**: `transaction_analysis`

### フィルター設定

| フィルター名 | フィールド | 種別 | 備考 |
|------------|----------|------|------|
| 購買期間 | `order_date` | 日付範囲 | デフォルト: 過去30日 |
| 店舗エリア | `store_area` | ドロップダウン | 複数選択可 |

### 帳票一覧

| No | 帳票名 | グラフ種別 | 設定 |
|----|-------|----------|------|
| 1 | 売上金額 | カウンター | 下記参照 |
| 2 | 注文件数 | カウンター | 下記参照 |
| 3 | 客単価 | カウンター | 下記参照 |
| 4 | 新規・既存売上比率 | 円グラフ | 下記参照 |
| 5 | 日別売上推移 | 折れ線グラフ | 下記参照 |
| 6 | クラスター別売上 | 横棒グラフ | 下記参照 |

### 帳票詳細

#### 帳票1-3: KPIカウンター

| 帳票名 | 集計 | 表示形式 |
|-------|------|---------|
| 売上金額 | `SUM(subtotal)` | 通貨（万円） |
| 注文件数 | `COUNT(DISTINCT order_id)` | 数値 |
| 客単価 | `SUM(subtotal) / COUNT(DISTINCT order_id)` | 通貨（円） |

#### 帳票4: 新規・既存売上比率

| 項目 | 設定値 |
|-----|-------|
| グラフ種別 | 円グラフ |
| 値 | `customer_type` |
| 集計 | `SUM(subtotal)` |
| 色分け | 新規=オレンジ, 既存=青 |

**見るポイント**: 新規売上が10-20%なら健全、5%未満は新規獲得施策が必要

#### 帳票5: 日別売上推移

| 項目 | 設定値 |
|-----|-------|
| グラフ種別 | 折れ線グラフ |
| X軸 | `order_date` |
| Y軸 | `SUM(subtotal)` |
| 色分け | なし（シンプルに） |

**見るポイント**: 週末ピーク、急激な落ち込みの有無

#### 帳票6: クラスター別売上

| 項目 | 設定値 |
|-----|-------|
| グラフ種別 | 横棒グラフ |
| X軸 | `SUM(subtotal)` |
| Y軸 | `behavior_cluster_name` |
| ソート | 売上降順 |
| 色分け | 固定色 |

**見るポイント**: 週末まとめ買い族が最大貢献しているか

**クラスター別特徴（データ根拠）**:
- 週末まとめ買い族: バスケットサイズ15-30点、土日集中
- 平日夕方の時短族: 即食系購買率90%（惣菜30%+冷凍30%+パン30%）
- 健康・こだわり層: オーガニック購買率84%（野菜28%+乳製品28%+調味料28%）
- 夜型値引きハンター: 値引き商品購買率75%、19-21時集中

**目的**: 売上の「なぜ」を深掘り、パターン発見

**データソース**: `transaction_analysis`

### フィルター設定

| フィルター名 | フィールド | 種別 | 備考 |
|------------|----------|------|------|
| 購買期間 | `order_date` | 日付範囲 | |
| 店舗エリア | `store_area` | ドロップダウン | |
| 店舗名 | `store_name` | ドロップダウン | |
| RFMセグメント | `rfm_segment` | ドロップダウン | |
| 時間帯 | `time_period` | ドロップダウン | |
| 曜日 | `order_dayofweek_name` | ドロップダウン | |

### 帳票一覧

| No | 帳票名 | グラフ種別 | 設定 |
|----|-------|----------|------|
| 1 | 日別売上推移（新規/既存） | 積み上げ棒グラフ | 下記参照 |
| 2 | 時間帯×曜日ヒートマップ | ヒートマップ | 下記参照 |
| 3 | カテゴリ別売上 | ドーナツグラフ | 下記参照 |
| 4 | 曜日別売上 | 棒グラフ | 下記参照 |

### 帳票詳細

#### 帳票1: 日別売上推移（新規/既存）

| 項目 | 設定値 |
|-----|-------|
| グラフ種別 | 積み上げ棒グラフ |
| X軸 | `order_date` |
| Y軸 | `SUM(subtotal)` |
| 色分け | `customer_type`（新規=オレンジ, 既存=青） |

**見るポイント**:
- 新規（オレンジ）の増減トレンド
- 既存（青）の安定性

#### 帳票2: 時間帯×曜日ヒートマップ

| 項目 | 設定値 |
|-----|-------|
| グラフ種別 | ヒートマップ |
| X軸 | `order_dayofweek_name`（月〜日の順） |
| Y軸 | `time_period`（朝→夜の順） |
| 値 | `SUM(subtotal)` |
| 色 | 青系グラデーション（濃い=売上高） |

**見るポイント**:
- 土日×午前〜午後: 週末まとめ買い族（バスケット15-30点）
- 平日×夕方(16-18時): 時短族（即食系90%、離反臨界点10日で要注意）
- 全曜日×夜(19-21時): 値引きハンター（値引き商品75%）

#### 帳票3: カテゴリ別売上

| 項目 | 設定値 |
|-----|-------|
| グラフ種別 | ドーナツグラフ |
| 値 | `category_name` |
| 集計 | `SUM(subtotal)` |
| ソート | 売上降順 |

**見るポイント**: 上位3カテゴリで50%以上を占めるか

#### 帳票4: 曜日別売上

| 項目 | 設定値 |
|-----|-------|
| グラフ種別 | 棒グラフ |
| X軸 | `order_dayofweek_name`（月〜日の順） |
| Y軸 | `SUM(subtotal)` |
| 色分け | `is_weekend`（TRUE=オレンジ, FALSE=青） |

**見るポイント**: 土日（オレンジ）が平日の1.5倍以上あるか

**見るポイント**: ランダム層が60%、特徴クラスターが40%

**見るポイント**: 上位20%（デシル1-2）が売上の40%を占めるか

**見るポイント**:

## 6. クロスフィルタリング活用例

### 経営サマリー → 売上分析への深掘り

1. **クラスター別売上**で「週末まとめ買い族」をクリック
2. → 売上分析タブに移動
3. → 時間帯×曜日ヒートマップで土日午前に集中していることを確認

### 売上分析での発見

1. **時間帯×曜日ヒートマップ**で「土曜×午後」をクリック
2. → 他のグラフが全て土曜午後に絞り込まれる
3. → カテゴリ別売上で「野菜・果物」が多いことが判明
4. → 施策: 土曜午後に野菜の特売セール

### 顧客分析での施策立案

1. **RFMセグメント分布**で「離脱リスク」をクリック
2. → リピートステータス分布が離脱リスク顧客のみに絞り込まれる
3. → 「時短族」の離脱リスクが多いことが判明
4. → 施策: 時短族向け「惣菜・冷凍食品セット緊急クーポン」配信（即食系購買率90%のため）
