# テーブル定義書

## 1. メダリオンアーキテクチャ概要

### レイヤーの定義

| レイヤー | 目的 | 特徴 |
|---------|------|------|
| **Bronze** | Rawデータの保持 | ・元データをそのまま保持（データ型変換なし）<br>・全カラムをSTRING型で取り込み<br>・監査カラム（_ingested_at, _source_file）を追加 |
| **Silver** | クレンジング + 非正規化 | ・データ型の変換（STRING → DATE, BIGINT等）<br>・NULL値の処理、不正データの修正<br>・重複排除<br>・非正規化（store_id + store_name 共存など） |
| **Gold** | 分析セグメント付与 | ・粒度はSilverと同じ（集計しない）<br>・RFMスコア、セグメント分類をカラムとして付与<br>・ダッシュボードのクロスフィルタリングに最適化 |

### テーブル一覧

| レイヤー | テーブル名 | 粒度 | 説明 |
|---------|-----------|------|------|
| **Bronze** | bz_users | 会員 | 会員マスタ（Raw） |
| | bz_items | 商品 | 商品マスタ（Raw） |
| | bz_stores | 店舗 | 店舗マスタ（Raw） |
| | bz_orders | 会計 | POS注文（Raw） |
| | bz_order_items | アイテム | POS注文明細（Raw） |
| **Silver** | sl_users | 会員 | 会員マスタ（クレンジング済み） |
| | sl_items | 商品 | 商品マスタ（クレンジング済み） |
| | sl_stores | 店舗 | 店舗マスタ（クレンジング済み） |
| | sl_orders | 会計 | POS注文（クレンジング + 非正規化） |
| | sl_order_items | アイテム | POS注文明細（クレンジング + 非正規化） |
| **Gold** | gd_users | 会員 | 会員 + 分析セグメント付与 |
| | gd_orders | 会計 | POS注文 + 分析セグメント付与 |
| | gd_order_items | アイテム | POS注文明細 + 分析セグメント付与 |

## 4. Goldテーブル

### gd_users（会員・分析セグメント付与済み）
会員マスタにRFMスコア・セグメント分類を付与。**行動クラスターは購買履歴から算出**。

| カラム名 | データ型 | 説明 |
|---------|---------|------|
| user_id | STRING | 会員ID |
| name | STRING | 氏名 |
| gender | STRING | 性別（M=男性, F=女性） |
| birth_date | DATE | 生年月日 |
| pref | STRING | 都道府県 |
| phone_number | STRING | 電話番号 |
| email | STRING | メールアドレス |
| email_permission_flg | BOOLEAN | メール配信許諾フラグ（true=許諾, false=拒否） |
| registration_date | DATE | 会員登録日 |
| status | STRING | 会員ステータス（active=有効, inactive=退会・停止） |
| last_login_at | TIMESTAMP | 最終ログイン日時 |
| behavior_cluster | STRING | 行動クラスター（1=週末まとめ買い族, 2=夜型値引きハンター, 3=健康・こだわり層, 4=カテゴリジャンキー, 5=平日夕方の時短族, null=ランダム） |
| behavior_cluster_name | STRING | 行動クラスター名（週末まとめ買い族/夜型値引きハンター/健康・こだわり層/カテゴリジャンキー/平日夕方の時短族/ランダム） |
| last_purchase_date | DATE | 最終購買日 |
| recency | INT | 最終購買からの経過日数（小さいほど最近購入） |
| frequency | INT | 購買回数（累計注文回数） |
| monetary | BIGINT | 累計購買金額（顧客単価ではなく累計金額） |
| r_score | INT | Recencyスコア（5=7日以内, 4=14日以内, 3=30日以内, 2=60日以内, 1=60日超） |
| f_score | INT | Frequencyスコア（5=10回以上, 4=5-9回, 3=3-4回, 2=2回, 1=1回） |
| m_score | INT | Monetaryスコア（5=5万円以上, 4=2-5万円, 3=1-2万円, 2=5千-1万円, 1=5千円未満） |
| rfm_segment | STRING | RFMセグメント（ロイヤル顧客/常連顧客/新規顧客/離脱リスク/休眠顧客/一般顧客） |
| customer_type | STRING | 顧客区分（新規=直近1ヶ月内に初回購入, 既存=それ以外） |

#### behavior_cluster の算出ロジック
購買履歴から以下の条件で自動判定:

| クラスター | 判定条件 |
|-----------|---------|
| 1: 週末まとめ買い族 | 土日購入率 ≥ 80% AND 平均バスケットサイズ ≥ 10点 |
| 2: 夜型値引きハンター | 19時以降購入率 ≥ 70% AND 値引き商品購入率 ≥ 50% |
| 3: 健康・こだわり層 | オーガニック/プレミアム商品購入率 ≥ 60% |
| 4: カテゴリジャンキー | 特定カテゴリ（酒/菓子/スイーツ）購入率 ≥ 70% |
| 5: 平日夕方の時短族 | 平日16-18時購入率 ≥ 70% AND 即食系購入率 ≥ 60% |
| NULL: ランダム層 | 上記いずれにも該当しない |
