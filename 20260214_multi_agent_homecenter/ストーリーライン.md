# Part 1: ダッシュボード + Genie デモ

## Step 1: ダッシュボードの表現力

> ダッシュボードでどんな表現ができるかを見せる

### チャートの種類（表現ページ）

| チャート種類 | 用途 |
|-------------|------|
| サンキーチャート | 関係性の可視化 |
| 箱ひげ図 | 出荷モード別の売上分布 |
| エリアグラフ | 時系列の推移 |
| カウンター（予実） | 売上 vs 予算の達成率 |
| 二重軸チャート | 異なるスケールの指標を同時比較 |
| ヒートマップ | 顧客区分 × 地域のクロス分析 |
| 散布図 | 売上と利益の相関 |
| 円グラフ | 構成比 |
| MAP（コロプレス） | 地理データの可視化 |

### AI機能（折れ線グラフ / 時系列チャートから実行）

| 機能 | 操作 | Wow ポイント |
|------|------|-------------|
| **予測モード** | 折れ線グラフのメニューから「予測」を実行 | 過去データからトレンドを自動予測し、将来の見通しを可視化 |
| **Explain Increase** | 上昇している区間を選択 →「増加を説明」 | なぜ増えたのか、AIが要因を自動分析して説明 |
| **Explain Decrease** | 下降している区間を選択 →「減少を説明」 | なぜ減ったのか、AIが要因を自動分析して説明 |

# Part 2: Agent Bricks マルチエージェントスーパーバイザー

> **テーマ**: 店舗スタッフの現場業務支援 — 複数AIエージェントが連携して一発回答

### 構成エージェント

| タイプ | 名前 | 役割 |
|--------|------|------|
| UC関数 | `get_current_staff_info` | ログインユーザーの従業員情報を特定 |
| Genie | ホームセンター在庫分析 | 構造化データ（在庫・商品・予約・売上・従業員）の検索・分析 |
| ナレッジアシスタント | kw-store-knowledge-bot | 社内マニュアル・研修資料・FAQ・商品知識の参照 |
| MCPサーバー | tavily_search | Web上の最新情報を検索 |

### デモ質問（連続会話）

| # | 質問 | 使われるエージェント | 狙い・意図 |
|---|------|----------------------|-----------|
| 1 | 受付済みの取り置き予約を確認して | UC関数 + Genie | UC関数でログインユーザーを特定 → Genieで自分の担当予約だけ自動フィルタ。Genie単体では複数ラウンド必要な操作がスーパーバイザーなら一発で完結する。 |
| 2 | 佐藤工務店様の予約、確保できていますか？ | Genie + RAG | Q1の結果を受けて、Genieで販売可能在庫を確認し、RAGで安全在庫基準・プロ会員例外条件を参照。数値＋ルール統合判断。 |
| 3 | このお客様、ウッドデッキも作りたいらしい。多摩店で資材揃う？ | RAG → Genie | RAGがDIY相談フローから必要資材を判断 → Genieが多摩店の在庫を一括チェック → 不足分の代替案まで出せるか。**★最大の見せ場。** |
| 4 | ウッドデッキDIYの初心者向けの手順やコツをWebで調べてまとめて | Tavily | 社内データにない外部情報をWeb検索で取得し、お客様への提案に活用できるか確認する。 |
